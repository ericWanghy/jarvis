# Intent Instruction: MEMORY_SCAN

用户主要意图：系统后台任务，扫描对话日志并提取有价值的记忆片段。

## 目标
从给的对话日志中，提取出值得长期保存的信息，特别是用户的长期目标、行动计划、偏好和关键事实。忽略闲聊和临时性问答。

## 提取规则
1. **Facts (事实)**: 用户明确陈述的客观事实（职业、技能、设备、家庭成员）。
2. **Preferences (偏好)**: 用户表达的好恶（喜欢深色模式，讨厌啰嗦）。**必须包含上下文**（例如：“喜欢红色” -> “喜欢红色作为 UI 主题色”）。
3. **Goals (目标)**: 用户的长期愿景、学习计划或职业目标（例如：“今年想精通 Rust”）。
4. **Workflows (工作流)**: 用户的习惯性工作流程或周期性任务（例如：“每周一需要生成周报”）。
5. **Projects (项目)**: 用户正在进行的项目及其当前状态。
6. **Insights (洞察)**: 从用户行为中推断出的潜在需求或思维模式。

## 过滤标准
- 忽略：打招呼、简单的 API 调用测试、已知的系统指令。
- 忽略：过于琐碎的细节（如“刚才吃了什么”）。
- 忽略：低置信度的推测。

## 输出格式（JSON）
**重要：content 字段必须使用中文描述。**

Strict JSON array of objects:
```json
[
  {
    "content": "用户偏好简洁的回答，不喜欢废话。",
    "category": "preference"
  },
  {
    "content": "用户正在使用 Tauri 和 Python 开发 Jarvis v5.4。",
    "category": "project"
  },
  {
    "content": "用户计划在今年内掌握 Rust 语言。",
    "category": "goal"
  }
]
```
若无有价值信息，返回空数组 `[]`。
