================================================================================
TROUBLESHOOTING
================================================================================

If tests fail, check:

1. Network Connection:
   - Can you reach api-hub.inner.chj.cloud?
   - Is there a firewall blocking port 80?
   - Check your network connectivity

2. Configuration:
   - Is app/core/config.py correct?
   - GEMINI_BASE_URL should be: http://api-hub.inner.chj.cloud/llm-gateway/v1
   - GEMINI_MODEL should be: gemini-3-pro-preview
   - GEMINI_GW_TOKEN should be set

3. Virtual Environment:
   - Did you activate venv? (source venv/bin/activate)
   - Are all dependencies installed? (pip install -r requirements.txt)

4. Gateway Issues:
   - Is the LLM gateway running and accessible?
   - Check gateway logs for errors
   - Verify token hasn't expired

5. Debug Logs:
   - Check gemini_debug.log for detailed error info
   - Look for HTTP error codes or connection timeouts
   - Review exception stack traces if present

================================================================================
CUSTOMIZING THE TESTS
================================================================================

To test a different query:

1. Edit test_gemini_debug.py or test_gemini_debug_extended.py
2. Find the Message creation line (search for "Message(")
3. Change the content parameter to your custom query
4. Save and run the script

Example:
  Before: content="Hello, can you say 'Hello' back to me?"
  After:  content="Explain quantum computing in one sentence"

USING LOGS FOR DEBUGGING
================================================================================

The generated logs contain valuable information:

- Timestamps: Exact when each operation occurred
